# База данных PcStressTester

База данных хранится в SQLite-файле:

```text
C:\Users\serge\Desktop\diplom\test\PcStressTester\Data\pc_stress_tester.db
```

При запуске из проекта приложение автоматически находит папку `PcStressTester` и использует файл из `Data`. Если приложение будет опубликовано без файла `PcStressTester.csproj`, база создастся рядом с исполняемым файлом в папке `Data`.

## Что сохраняется

`test_runs` - запуски стресс-тестов CPU/GPU: тип теста, время начала и завершения, целевая нагрузка, статус и итоговое описание.

`sensor_snapshots` - основные агрегированные показания датчиков на момент обновления интерфейса: температура и загрузка CPU/GPU, частота CPU, использование RAM и текущий статус приложения.

`sensor_values` - подробные значения всех датчиков LibreHardwareMonitor, связанные с конкретным снимком из `sensor_snapshots`.

`test_metrics` - итоговые метрики теста, например максимальная температура CPU, пиковая и средняя загрузка, длительность и число потоков.

Для просмотра на русском в схеме есть представления `запуски_тестов`, `снимки_датчиков`, `значения_датчиков` и `метрики_тестов`. Основные таблицы оставлены с техническими английскими именами для совместимости с кодом приложения и уже созданной базой.

## Как использовать

1. Запустить приложение.
2. Нажать запуск CPU- или GPU-теста.
3. Во время работы приложение будет сохранять снимки датчиков примерно раз в секунду.
4. После остановки или завершения теста запись в `test_runs` получит русский статус `Завершен` или `Остановлен`, а итоговые показатели CPU-теста попадут в `test_metrics` с русскими названиями метрик.

## Примеры SQL-запросов

Последние запуски тестов:

```sql
SELECT *
FROM "запуски_тестов"
ORDER BY "начало" DESC;
```

Последние 50 снимков датчиков:

```sql
SELECT *
FROM "снимки_датчиков"
ORDER BY "время_снимка" DESC
LIMIT 50;
```

Все подробные датчики для выбранного снимка:

```sql
SELECT *
FROM "значения_датчиков"
WHERE "идентификатор_снимка" = 1
ORDER BY "устройство", "тип_датчика", "датчик";
```

Итоговые метрики выбранного теста:

```sql
SELECT "метрика", "значение", "единица_измерения"
FROM "метрики_тестов"
WHERE "идентификатор_теста" = 1
ORDER BY "метрика";
```

Открыть файл можно любой SQLite-программой, например DB Browser for SQLite, DBeaver или встроенным SQLite-плагином IDE.
